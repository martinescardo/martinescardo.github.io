Martin Escardo 14 May 2014

Function extensionality follows from a single equation.

Fix a universe U, say Set₀ in Agda, but this informal discussion
preceding the Agda code is in MLTT.

For X : U and Y : X → U, define

  funext X Y  = Π(f g : Π X Y). (Π(x : X). f x = g x) → f = g,

       Funext = Π(X : U)(Y : X → U). funext X Y. 

The following is proved in Martin-Löf type theory without K and without
any homotopical extension. 

   Theorem
   -------
      For any given type X : U and family Y : X → U, we can build a
      type I : U with two points i₀, i₁ : I such that 

           i₀ = i₁ → funext(X,Y).

      Conversely, from Funext one gets that the constructed type I is
      an hproposition and hence that i₀ = i₁ holds.

The idea of the proof is that, for any x : X, the type Y x believes
that the type I constructed in the theorem is an interval, even
judgementally, which is important to prove the theorem. (We also
require the eta-law. Without it, as usual, we conclude that the
eta-expansions of two pointwise equal functions are equal.)

The point is that we construct this relative-interval without assuming
higher-inductive types, not even hpropositional truncations. But this
is inspired by the proof that the hpropositional truncation of the
two-point type, if available, is the interval, and Mike Shulman's
proof that an interval implies funext.

A more explicit way of formulating the above is:

   Theorem'
   -------
     The type Funext is logically equivalent to (the hproposition)

       Π(X : U)(Y : X → U). (λ x y u. u 0) = (λ x y u. u 1),

     where the identity is in the type

       Π(x : X)(y : Y x). (2 → Σ(y': Y). y = y') → Σ(y': Y). y = y' 

     and 2 is the two-point type with elements 0,1.

\begin{code}

{-# OPTIONS --without-K #-}

open import tiny-library 

U = Set

module funext-from-a-sigle-equation (X : U) (Y : X → U) where

 I : U
 I = ∀(x : X) (y : Y x) → (₂ → paths-from y) → paths-from y

 i₀ i₁ : I
 i₀ x y u = u ₀
 i₁ x y u = u ₁

 I-rec' : {x : X} {y₀ y₁ : Y x} → y₀ ≡ y₁ → I → paths-from y₀
 I-rec' {x} {y₀} {y₁} p i = i x y₀ u
  where
   u : ₂ → paths-from y₀
   u ₀ = y₀ , refl -- = trivial-loop y₀
   u ₁ = y₁ , p

 I-rec : {x : X} {y₀ y₁ : Y x} → y₀ ≡ y₁ → I → Y x
 I-rec p i = end-point(I-rec' p i)

 theorem : i₀ ≡ i₁ → ∀(f g : (x : X) → Y x) → (∀(x : X) -> f x ≡ g x) → f ≡ g
 theorem seg f g h = ap ψ seg 
  where
   φ : (x : X) → I → Y x
   φ x = I-rec (h x) 
   ψ : I → (x : X) → Y x
   ψ i x = φ x i

\end{code}

This proof relies on the following judgemental equalities:

\begin{code} 

 equation₀ : ∀ {x : X} {y₀ y₁ : Y x} (p : y₀ ≡ y₁) → I-rec p i₀ ≡ y₀
 equation₀ p = refl 

 equation₁ : ∀ {x : X} {y₀ y₁ : Y x} (p : y₀ ≡ y₁) → I-rec p i₁ ≡ y₁
 equation₁ p = refl

\end{code}

If we replace the definition of I-rec by a postulate, the proof of the
theorem doesn't type-check, and the last two equations don't hold
judgementally.

\begin{code} 

 equation₂ : ∀ (seg : i₀ ≡ i₁) {x : X} {y₀ y₁ : Y x} (p : y₀ ≡ y₁) → ap (I-rec p) seg ≡ p
 equation₂ seg {x} {y} refl = r ⁻¹ • q
  where
   p : ap (I-rec' refl) seg ≡ refl 
   p = paths-from-is-hset y (I-rec' refl i₀) (trivial-loop y) (ap (I-rec' refl) seg) refl
   q : ap end-point (ap (I-rec' refl) seg) ≡ refl
   q = ap (ap end-point) p
   r : ap end-point (ap (I-rec' refl) seg) ≡ ap (I-rec refl) seg
   r = ap-functorial (I-rec' refl) end-point seg

\end{code}

(We can do something similar with the judgemental factorization of
hsetfunext.lagda.)

Here is a concise and cryptic version that avoids the type ₂:

\begin{code} 

module concise (X : U) (Y : X → U) where

 I : U
 I = (x : X) (y : Y x) → paths-from y → paths-from y → paths-from y

 i₀ i₁ : I
 i₀ x y u₀ u₁ = u₀
 i₁ x y u₀ u₁ = u₁

 I-rec : (x : X) (y₀ y₁ : Y x) → y₀ ≡ y₁ → I → Y x
 I-rec x y₀ y₁ p i = π₀(i x y₀ (y₀ , refl) (y₁ , p))

 theorem : i₀ ≡ i₁ → ∀(f g : (x : X) → Y x) → (∀(x : X) -> f x ≡ g x) → f ≡ g
 theorem seg f g h = ap (λ i x → I-rec x (f x) (g x) (h x) i) seg 

\end{code}

Or, β-expanding:

\begin{code} 

module too-concise (X : U) (Y : X → U) where

 I : U
 I = (x : X) (y : Y x) → paths-from y → paths-from y → paths-from y

 i₀ i₁ : I
 i₀ x y u₀ u₁ = u₀
 i₁ x y u₀ u₁ = u₁

 theorem : i₀ ≡ i₁ → ∀(f g : (x : X) → Y x) → (∀(x : X) -> f x ≡ g x) → f ≡ g
 theorem seg f g h = ap (λ i x → π₀(i x (f x) (f x , refl) (g x , h x))) seg

\end{code}

Now we can see clearly why what one actually gets directly is the
equation (λ x → f x) ≡ (λ x → g x). But Agda silently η-contracts each
side to accept f ≡ g.

It is not worth further β-expanding i₀ and i₁, because then we (and
Agda) won't know their types without giving them in their
λ-expressions. Nevertheless, here it goes. It suffices to give the
type of one variable. We don't need to define the type I either, but
of course it occurs, as an inferred type. 

\begin{code}

module ultimately-concise (X : U) (Y : X → U) where

 theorem : (λ x y (u₀ : paths-from y) u₁ → u₀)  ≡  (λ x y u₀ u₁ → u₁) 
         → ∀(f g : (x : X) → Y x) → (∀(x : X) -> f x ≡ g x) → f ≡ g
 theorem seg f g h = ap (λ i x → π₀(i x (f x) (f x , refl) (g x , h x))) seg

\end{code}

This illustrates how not to formulate and prove a theorem.

But we did get a small theorem with a proof shorter than its formulation.

I would settle for this if I wanted to be really concise but not
completely obscure:

\begin{code} 

module parsimoniously-concise (X : U) (Y : X → U) where

 I : U
 I = (x : X) (y : Y x) → paths-from y → paths-from y → paths-from y

 theorem : (λ x y u v → u) ≡ (λ x y u v → v) 
         → ∀(f g : (x : X) → Y x) → (∀(x : X) -> f x ≡ g x) → f ≡ g
 theorem seg f g h = ap φ seg
  where
   φ : I → (x : X) → Y x
   φ i x = π₀(i x (f x) (f x , refl) (g x , h x))

   observation₀ : φ(λ x y u v → u) ≡ f
   observation₀ = refl

   observation₁ : φ(λ x y u v → v) ≡ g
   observation₁ = refl

\end{code}
